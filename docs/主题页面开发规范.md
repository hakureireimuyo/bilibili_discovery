# 主题页面开发规范

## 目标

这份文档用于统一后续页面的主题接入方式，解决以下问题：

1. 新页面如何接入主题系统
2. 页面样式如何与预设主题交互
3. 什么时候可以新增主题变量
4. 如何避免样式代码越来越臃肿、失控

本文档适用于 `src/ui` 下所有新页面，也适用于旧页面重构。

## 核心原则

主题系统分成两层：

1. 主题层
   负责提供稳定的设计令牌，也就是 CSS 变量，例如文本色、背景色、边框色、按钮色、阴影色。
2. 页面层
   负责定义页面结构、布局、组件组合和交互状态，但不直接写死配色。

最终关系是：

`主题配置 -> 主题变量 -> 页面 CSS 使用变量 -> 页面渲染`

页面不应该直接知道某个主题的具体颜色值，只应该使用 `var(--theme-*)`。

## 新页面接入主题的标准流程

### 1. 创建页面文件

一个新页面通常包含：

- `xxx.html`
- `xxx.css`
- `xxx.ts`

例如：

```text
src/ui/my-page/
├── my-page.html
├── my-page.css
└── my-page.ts
```

### 2. HTML 使用模块脚本

页面脚本必须使用模块加载：

```html
<script type="module" src="./my-page.ts"></script>
```

不要使用普通 `<script src="...">` 去加载包含 `import/export` 的脚本。

### 3. 在页面入口初始化主题

在页面入口文件中调用 `initThemedPage(...)`：

```ts
import { initThemedPage } from "../../themes/index.js";

export function initMyPage(): void {
  if (typeof document === "undefined") {
    return;
  }

  initThemedPage("options");
}
```

如果是新页面，请先在 [page-theme.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/page-theme.ts) 的 `ThemePageId` 里加上新的页面 ID。

推荐页面 ID 命名：

- `popup`
- `options`
- `stats`
- `my-page`

命名要短、稳定、语义明确。

### 4. CSS 只能消费主题变量

页面 CSS 必须通过 `var(--theme-*)` 获取颜色、背景、边框、阴影。

正确示例：

```css
body {
  background: var(--theme-page-bg);
  color: var(--theme-text-primary);
}

.card {
  background: var(--theme-surface-base);
  border: 1px solid var(--theme-border-primary);
  box-shadow: 0 8px 24px var(--theme-shadow-light);
}
```

错误示例：

```css
.card {
  background: #ffffff;
  border: 1px solid #e5e7eb;
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.08);
}
```

## 页面与主题如何交互

### 静态样式交互

最常见的方式是页面 CSS 直接使用主题变量：

```css
.toolbar {
  background: var(--theme-surface-muted);
  color: var(--theme-text-secondary);
}
```

页面本身不关心当前是蓝色主题还是莫兰迪主题，只关心这里是一个“工具栏”。

### 动态逻辑交互

如果页面有图表、canvas、自定义渲染、运行时动画等不能只靠 CSS 完成的场景，可以通过主题管理器获取当前主题：

```ts
import { themeManager } from "../../themes/index.js";

const theme = themeManager.getCurrentTheme();
```

例如 `stats` 页的 tag 颜色就不是写死在页面里，而是通过 `theme.tagColors` 决定映射范围，再由工具函数按字符串稳定计算颜色。

如果页面需要在切换主题后更新运行时内容，可以监听主题变化：

```ts
import { themeManager } from "../../themes/index.js";

const listener = (theme) => {
  // 更新图表颜色、重新绘制 canvas 等
};

themeManager.addChangeListener(listener);
```

如果页面注册了监听器，销毁时必须移除：

```ts
themeManager.removeChangeListener(listener);
```

### 跨页面更新

主题管理器已经负责：

- 当前页面变量注入
- `localStorage` 持久化
- 跨页面同步通知

页面不需要自己处理“其他页面切换主题后我怎么更新”这类问题，只要通过主题系统接入即可。

## 什么时候可以新增主题变量

新增变量前，必须先判断属于哪一类。

### 第一类：通用语义变量

如果这个样式语义可能被多个页面共享，就应该加到通用语义层。

例如：

- `--theme-page-bg`
- `--theme-surface-base`
- `--theme-text-primary`
- `--theme-border-primary`
- `--theme-button-primary-bg`

适合场景：

- 页面背景
- 卡片背景
- 全局按钮
- 输入框边框
- 通用 hover/focus 状态

### 第二类：页面域变量

如果这个变量只属于某个页面稳定存在的视觉语义，就加页面域变量。

例如 stats 页：

- `--theme-stats-drop-bg`
- `--theme-stats-name-line`

例如 popup 页：

- `--theme-popup-shell-bg`
- `--theme-popup-status-bg`

页面域变量必须满足：

1. 这个需求无法用通用语义变量表达
2. 这个需求是页面内稳定存在的，不是临时一次性样式
3. 命名必须带页面前缀

## 新增主题变量的流程

### 步骤 1

先检查 [theme-variables.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/theme-variables.ts) 中已有变量能否复用。

### 步骤 2

如果不能复用，再决定新增通用变量还是页面域变量。

### 步骤 3

在 [theme-variables.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/theme-variables.ts) 中统一新增派生逻辑。

示例：

```ts
"--theme-my-page-highlight-bg": toRgba(colors.primary, 0.12)
```

### 步骤 4

页面 CSS 改为消费新变量：

```css
.highlight-panel {
  background: var(--theme-my-page-highlight-bg);
}
```

### 步骤 5

构建并确认所有主题都能正常显示。

## 新增主题配置的流程

新增主题统一由开发者在代码中维护，不开放用户侧创建。

### 步骤 1

在 [types.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/types.ts) 的 `ThemeId` 中增加新的主题 ID。

### 步骤 2

在 [theme-configs.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/theme-configs.ts) 中补充该主题的浅色和深色配置。

### 步骤 3

除了 `colors` 外，必须同步补齐 `tagColors`：

```ts
tagColors: {
  hueStart: 0,
  hueRange: 360,
  saturationMin: 40,
  saturationMax: 54,
  lightnessMin: 85,
  lightnessMax: 94
}
```

### 步骤 4

将新主题加入 `themeConfigs` 导出数组，并执行构建验证。

## 严格禁止的写法

以下写法默认禁止，除非有非常明确的理由。

### 1. 在页面 CSS 中直接写死主题颜色

禁止：

```css
color: #1f2430;
background: #ffffff;
```

### 2. 在 TS 中直接拼视觉颜色

禁止：

```ts
element.style.backgroundColor = "#8b9dc3";
```

允许的例外：

- 标签颜色按内容动态计算
- canvas 绘制临时色值
- 第三方库必须传入颜色字符串

但即使是例外，也优先从主题变量或主题配置中取值。

### 3. 内联样式承载主题逻辑

禁止：

```html
<a style="color: #8b9dc3">
```

应该改为 class + CSS 变量。

### 4. 页面自己管理主题状态

禁止页面各自维护：

- 当前主题名
- 本页 CSS 变量注入
- 主题持久化逻辑
- 跨页面同步逻辑

这些必须统一交给主题系统。

## 推荐写法

### 布局与结构

页面 CSS 负责：

- grid / flex 布局
- 间距
- 圆角
- 字号
- 组件结构

页面 CSS 不负责定义具体主题配色值。

### 状态切换

优先使用 class 切换状态，而不是在 TS 里写一堆 `style.xxx`。

推荐：

```ts
element.classList.add("is-active");
```

```css
.panel.is-active {
  background: var(--theme-surface-accent);
}
```

不推荐：

```ts
element.style.background = "rgba(...)";
```

### 动态效果

像宽度、位移、透明度这类纯交互数据可以继续放在运行时样式里：

- `style.width`
- `style.transform`
- `style.opacity`

因为这类值通常不是主题设计令牌，而是即时交互状态。

## 新页面创建模板

### HTML

```html
<!doctype html>
<html lang="zh-CN">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>My Page</title>
    <link rel="stylesheet" href="./my-page.css" />
  </head>
  <body>
    <div class="app">
      <header class="page-header">
        <h1>My Page</h1>
      </header>
      <main class="page-content">
        <section class="card">内容区域</section>
      </main>
    </div>
    <script type="module" src="./my-page.ts"></script>
  </body>
</html>
```

### TS

```ts
import { initThemedPage } from "../../themes/index.js";

export function initMyPage(): void {
  if (typeof document === "undefined") {
    return;
  }

  initThemedPage("my-page");
}

if (typeof document !== "undefined") {
  initMyPage();
}
```

### CSS

```css
body {
  margin: 0;
  background: var(--theme-page-bg);
  color: var(--theme-text-primary);
}

.app {
  min-height: 100vh;
  padding: 24px;
}

.card {
  background: var(--theme-surface-base);
  border: 1px solid var(--theme-border-primary);
  box-shadow: 0 8px 24px var(--theme-shadow-light);
  border-radius: 16px;
  padding: 16px;
}
```

## Code Review 检查清单

新增页面或重构页面时，Review 必查以下内容：

1. 页面入口是否调用了 `initThemedPage(...)`
2. HTML 是否使用 `type="module"`
3. CSS 是否使用 `var(--theme-*)`
4. 是否存在硬编码颜色
5. 是否存在主题相关内联样式
6. 是否存在页面自己处理主题持久化或同步
7. 新增变量是否真的有必要，是否能复用已有 token
8. 新增页面域变量是否带页面前缀
9. 新主题是否补齐了 `tagColors`

## 后续维护建议

1. 所有新页面默认先复用现有 token，不要急着新增变量
2. 如果一个视觉语义在两个页面以上重复出现，再考虑升级为通用变量
3. 页面域变量控制在“必要且稳定”的范围内，不要把整个页面每个颜色都变成专属 token
4. 主题变量统一在 [theme-variables.ts](/D:/ProjectFloder/TS/bilibili_discovery/src/themes/theme-variables.ts) 维护，不要分散到各页面
5. 新增主题或重构主题时，优先保证语义一致，而不是追求每页单独定制
6. 涉及 tag 胶囊、拖拽预览等字符串映射颜色时，统一走主题里的 `tagColors` 配置，不要在页面里再写自己的哈希范围
