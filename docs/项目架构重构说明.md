# 项目层次结构说明书（严格版）

## 一、总体架构层次

```
src/database/indexeddb      → 底层驱动（通用 IndexedDB CRUD & index）
src/database/implementations → 数据访问层（高性能 CRUD + 最小查询）
src/database/types           → 数据结构定义（轻数据/重数据）

src/cache                    → 数据缓存层（跨页面共享、按需加载）
src/query                    → 查询引擎层（复杂查询、过滤、排序、分页）

src/ui / 页面层              → 仅调用 query 层，按需加载重数据
```

---

## 二、各层职责与能力边界

### 1️⃣ src/database/indexeddb（底层驱动层）

**核心职责：**

* 封装 IndexedDB 原生 API
* 提供通用 CRUD、index 查询、cursor 迭代

**提供能力：**

* `get(key)`
* `put(data)`
* `delete(key)`
* `getAll()`
* `getByIndex(indexName, value)`
* `cursor(callback)`

**禁止事项：**

* 任何业务逻辑
* Creator / Tag 等具体概念
* filter / sort / complex query

**调用方式：**

* 仅被 `implementations` 层调用

---

### 2️⃣ src/database/types（数据结构层）

**核心职责：**

* 定义所有数据结构
* 不包含逻辑

**结构示例：**

**轻数据结构 `Creator`**：

```ts
interface Creator {
  creatorId: ID;
  platform: Platform;
  name: string;
  avatarUrl: string;      // 轻量 URL 占位
  isLogout: number;
  followTime: Timestamp;
  isFollowing: number;
  tagWeights: CreatorTagWeight[];
  descriptionPreview?: string; // 可选轻量摘要
}
```

**重数据结构 `CreatorAsset`**：

```ts
interface CreatorAsset {
  creatorId: ID;
  platform: Platform;
  avatar?: string;        // base64/blob
  description?: string;   // 完整描述
  updatedAt: Timestamp;
}
```

**禁止事项：**

* 方法、计算属性、业务逻辑

---

### 3️⃣ src/database/implementations（数据访问层）

**核心职责：**

* 封装表访问（Creator / CreatorAsset）
* 提供最小查询能力
* 保证轻量与低内存

**允许方法示例：**

* `getById(id)`
* `getByIds(ids[])`
* `getByPlatform(platform)`
* `getByFollowStatus(isFollowing)`
* `getAll()` ⚠️ 高成本
* `getPartial(fields[])` ✅ 支持部分字段加载

**禁止事项：**

* 多条件组合查询（tag + keyword + category）
* 排序、分类逻辑、复杂 filter
* 返回重数据（默认仅返回轻数据）

**重数据访问：**

* `CreatorAssetRepository` 提供按需获取方法：

```ts
getAvatar(creatorId, platform): Promise<string | null>
getDescription(creatorId, platform): Promise<string | null>
```

**调用方式：**

* 被 `cache` 层调用

---

### 4️⃣ src/cache（缓存层）

**核心职责：**

* 缓存轻数据和按需缓存重数据
* 控制缓存粒度（全量/部分字段）
* 跨页面共享缓存
* 生命周期管理（引用计数 + 自动释放）

**缓存结构示例：**

```ts
class TableCache<T> {
  data: T[];
  fields: Set<string>;
  refCount: number;
  lastAccessTime: number;
}
```

**核心功能：**

1. 获取缓存

```ts
getCache(table, options: { fields?: string[], filter?: basicFilter }): Promise<TableCache>
```

2. 引用管理

```ts
retain(tableKey)
release(tableKey)
```

3. 自动释放

* `refCount = 0` → 延迟释放（如 30s）
* 超时未使用 → 清理

4. 字段升级

* 已有缓存 `{id, name}`
* 请求 `{id, name, tagWeights}` → 增量加载缺失字段并合并

**禁止事项：**

* 复杂查询（tag / category / keyword）
* 排序、分页

**调用方式：**

* 被 `query` 层调用

---

### 5️⃣ src/query（查询引擎层）

**核心职责：**

* 唯一处理复杂查询、过滤、排序、组合逻辑
* 对轻数据进行计算
* 不访问重数据

**标准接口示例：**

```ts
queryCreators(options: {
  platform?: string,
  isFollowing?: boolean,
  includeTags?: string[],
  excludeTags?: string[],
  keyword?: string,
  sortBy?: 'followTime',
  page?: number,
  pageSize?: number
}): Promise<Creator[]>
```

**内部执行流程：**

1. 请求 `cache` 获取数据
2. 在内存中执行 filter / sort / paginate
3. 返回结果给 UI

**禁止事项：**

* 直接访问 IndexedDB
* 调用 `DBUtils`

---

### 6️⃣ UI / 页面层

**核心职责：**

* 调用 `query` 层获取数据
* 渲染数据
* 按需触发重数据加载（avatar / description）

**重数据加载机制：**

* 列表页：只显示 `name + avatarUrl`
* 进入视口：触发 `loadAvatar(creatorId)`
* 详情页：进入页面 → 加载完整 `avatar + description`

**禁止事项：**

* 直接操作 `cache` 或 `implementations`
* 编写 filter / sort

---

## 三、轻重数据分离规则

| 数据类型 | 缓存   | 生命周期                    | 加载策略                                |
| ---- | ---- | ----------------------- | ----------------------------------- |
| 轻数据  | 全量缓存 | 页面打开 → 页面关闭释放           | 全量或分页加载                             |
| 重数据  | 按需缓存 | 页面可见 → UI 请求 → cache 缓存 | 延迟加载 / IntersectionObserver / 详情页加载 |

**判定标准**：

* > 10KB 或 base64/blob 或长文本

---

## 四、运行机制（简化版）

1. 页面打开

   * Cache 获取轻数据（全量或分页）
   * Repository 提供最小查询

2. 页面操作

   * Query 层对轻数据执行 filter / sort / paginate
   * 不访问重数据

3. 重数据访问

   * UI 发起请求 → Repository 获取 → cache 按需存储

4. 页面关闭

   * Cache 释放所有轻/重数据

## 七、严格执行约束

1. implementations 层仅提供数据访问，禁止业务逻辑
2. cache 层仅缓存数据，禁止查询操作
3. query 层是唯一允许处理 filter / sort / pagination 的地方
4. UI 不能直接操作数据库或缓存
5. 重数据只在 UI 层按需获取
6. 所有复杂逻辑必须集中在 query 层
7. 生命周期管理必须严格执行
