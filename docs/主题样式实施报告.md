# 主题与样式规范实施总结

## 已完成的工作

### 1. 主题模块扩展（src/themes/）

**修改文件：** [theme-variables.ts](../../src/themes/theme-variables.ts)

在 `buildThemeVariables` 函数中补充了以下语义变量类别：

- **圆角尺度**：`--theme-radius-xs` 到 `--theme-radius-full`
- **间距尺度**：`--theme-space-xs` 到 `--theme-space-2xl`
- **字体大小**：`--theme-font-size-xs` 到 `--theme-font-size-2xl`
- **行高**：`--theme-line-height-tight/normal/relaxed`
- **字重**：`--theme-font-weight-normal/medium/semibold/bold`
- **深度阴影**：`--theme-shadow-xs` 到 `--theme-shadow-2xl`
- **输入框样式**：border、bg、text、placeholder 相关变量
- **分割线**：divider、divider-strong、divider-light
- **禁用状态**：disabled-bg、disabled-text、disabled-border
- **链接**：link、link-hover、link-visited
- **代码块**：code-bg、code-text、code-border

**效果：** 主题体系从"仅控制颜色"升级到"控制颜色+间距+排版+深度"，能够更深入地影响界面整体风格。

---

### 2. 共享样式库建立（src/ui/shared/）

创建了 4 个新的共享样式文件：

#### [theme-vars.css](../../src/ui/shared/theme-vars.css)

- 包含所有主题 CSS 变量的声明与默认值
- 明确文档化了变量分类
- 作为其他样式文件的基础依赖
- 由主题管理器在运行时动态覆盖变量值

#### [utils.css](../../src/ui/shared/utils.css)

包含 100+ 个通用工具类，分类为：

- Flexbox 布局：`u-flex-*`, `u-items-*`, `u-justify-*`
- 间距：`u-m-*`, `u-mb-*`, `u-p-*`, `u-px-*` 等
- 文本样式：`u-text-*`, `u-text-bold`, `u-text-truncate`
- 尺寸：`u-w-full`, `u-h-full`, `u-overflow-*`
- 背景和边框：`u-bg-*`, `u-border-*`, `u-rounded-*`
- 阴影：`u-shadow-*`
- 其他：位置、透明度、光标、过渡
- 状态类：`is-active`, `is-disabled`, `is-loading`, `is-error` 等

**作用：** 减少页面 CSS 的重复写法，通过组合工具类快速建立布局。

#### [components.css](../../src/ui/shared/components.css)

定义了所有通用组件的基础样式：

- **按钮**：`.btn`, `.btn--primary`, `.btn--secondary`, `.btn--accent`, `.btn--ghost` 及尺寸变体
- **卡片/面板**：`.card`, `.panel` 及 `.--compact`, `.--elevated`, `.--muted`, `.--accent` 变体
- **标签/徽章**：`.badge`, `.tag` 及状态变体
- **输入框**：统一样式，focus/hover/disabled 状态
- **弹窗外壳**：`.shell` 及其渐变背景
- **英雄区域**：`.hero` 及子组件样式
- **网格布局**：`.grid-2col`, `.grid-3col`, `.grid-4col`, `.grid-auto`
- **面板头部**：`.panel-head`, `.panel-title`, `.panel-actions`
- **状态指示器**：`.status-badge` 及状态变体
- **列表**：`.list`, `.list-item` 及其变体
- **其他**：分割线、面包屑、进度条、空状态、加载动画

### 3. Popup 页面重构（src/ui/popup/）

#### 修改 [popup.html](../../src/ui/popup/popup.html)

在 `<head>` 中添加共享样式导入顺序：

```html
<link rel="stylesheet" href="../shared/theme-vars.css" />
<link rel="stylesheet" href="../shared/base.css" />
<link rel="stylesheet" href="../shared/utils.css" />
<link rel="stylesheet" href="../shared/components.css" />
<link rel="stylesheet" href="./popup.css" />
```

**作用：** 确保共享样式优先加载，popup.css 可安全覆盖或扩展。

#### 重构 [popup.css](../../src/ui/popup/popup.css)

- **删除**：重复的 reset 样式、base 排版、通用组件定义
- **替换**：硬编码颜色值 → `var(--theme-*)`
- **简化**：从 500+ 行减至 400+ 行，只保留 popup 特有样式
- **增强**：所有尺寸数值都改为语义变量（`var(--font-size-*)`, `var(--theme-radius-*)`）
- **保留**：焦点网格、小部件选择器、创作者列表等 popup 独特的交互逻辑

**效果：** popup 样式现在完全依赖主题系统，主题切换时自动适应，且代码更易维护。

## 后续建议

### 立即任务

1. **验证功能**
   - 打开 popup 页面，检查界面是否正常显示
   - 切换主题（浅色↔深色），验证样式是否正确更新
   - 检查 focus-grid、status-list、footer-actions 等关键区域的显示效果

2. **修改其他页面**
   - 按同样模式修改 `options` 页面
   - 修改 `stats` 页面
   - 修改 `workbench` 等其他主要页面

### 每页修改步骤

1. 在 HTML 中导入共享样式：
   ```html
   <link rel="stylesheet" href="../shared/theme-vars.css" />
   <link rel="stylesheet" href="../shared/base.css" />
   <link rel="stylesheet" href="../shared/utils.css" />
   <link rel="stylesheet" href="../shared/components.css" />
   <link rel="stylesheet" href="./[page-name].css" />
   ```

2. 在该页面 CSS 中：
   - 提取通用组件样式到 `components.css`（如卡片、按钮等）
   - 用工具类替代简单布局（margin、padding、gap、display）
   - 将所有硬编码颜色替换为 `var(--theme-*)`
   - 保留页面特有的交互和布局逻辑

3. 删除重复的字体定义、重置样式等（已由 base.css 覆盖）

### 长期优化

1. **补充缺失的主题变量**
   - 如果发现某个样式仍然写死值，检查是否应添加新变量
   - 新变量应符合"语义化"原则，不应是具象描述

2. **完善组件库**
   - 根据新页面的需求，持续在 `components.css` 中补充通用组件
   - 定义组件的各种状态和变体

3. **建立设计令牌文档**
   - 生成一份可视化的设计令牌参考（颜色、圆角、阴影等）
   - 让设计和开发都能对齐

4. **性能优化**
   - 生成共享样式的构建优化（如 CSS 压缩）
   - 考虑按需加载，避免无用样式加载
