
# 日志系统使用说明

## 概述

本项目实现了一个支持编译时模式切换的日志系统,可以根据构建模式自动控制日志输出。

## 日志级别

系统支持以下日志级别:
- `debug`: 调试信息
- `info`: 一般信息
- `warn`: 警告信息
- `error`: 错误信息
- `none`: 不输出任何日志

## 使用方法

### 1. 在代码中使用logger

```typescript
import { logger } from './utils/logger.js';

// 使用默认logger
logger.debug('调试信息');
logger.info('一般信息');
logger.warn('警告信息');
logger.error('错误信息');

// 创建带有前缀的子logger
const upCollectorLogger = logger.child('UpCollector');
upCollectorLogger.info('UP主信息已收集');
```

### 2. 构建模式

项目提供两种构建模式:

#### 开发模式 (Development)
- 日志级别: `debug`
- 输出所有级别的日志
- 使用命令: `npm run dev` 或 `npm run build:dev`

#### 生产模式 (Production)
- 日志级别: `none`
- 不输出任何日志
- 使用命令: `npm run release` 或 `npm run build:prod`

## 工作原理

1. **编译前处理**:
   - `prebuild.js` 脚本在编译前运行
   - 根据 `NODE_ENV` 环境变量设置日志级别
   - 修改 `src/env.ts` 文件中的占位符

2. **编译时注入**:
   - TypeScript 编译器将 `env.ts` 编译到输出目录
   - 日志级别在编译时被确定,而不是运行时

3. **运行时行为**:
   - Logger 根据编译时确定的日志级别过滤输出
   - 生产模式下,所有日志调用都会被优化掉

## 示例

### 开发模式构建

```bash
# 设置为开发模式并构建
npm run dev

# 或
npm run build:dev
```

输出示例:
```
[2024-03-28T10:30:00.000Z] [DEBUG] [UpCollector] 开始收集UP主数据
[2024-03-28T10:30:00.001Z] [INFO] [UpCollector] UP主信息已收集
```

### 生产模式构建

```bash
# 设置为生产模式并构建
npm run release

# 或
npm run build:prod
```

输出:
```
(无任何日志输出)
```

## 注意事项

1. 生产模式下,所有日志调用都会被优化掉,不会产生任何性能开销
2. 建议在开发时使用 `debug` 级别,在生产环境使用 `none` 级别
3. 日志级别在编译时确定,无法在运行时更改
4. 使用 `logger.child()` 创建带有特定前缀的子logger,便于追踪日志来源

## 迁移指南

如果你需要将现有的 `console.log` 调用迁移到新的日志系统:

1. 导入logger:
```typescript
import { logger } from './utils/logger.js';
```

2. 替换console调用:
```typescript
// 旧代码
console.log('[UpCollector] 开始收集UP主数据');

// 新代码
logger.child('UpCollector').debug('开始收集UP主数据');
```

3. 根据需要选择合适的日志级别:
   - `debug`: 详细的调试信息
   - `info`: 一般信息
   - `warn`: 警告信息
   - `error`: 错误信息
