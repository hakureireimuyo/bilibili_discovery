# 兴趣分析开发任务清单

本文档用于把 [`兴趣分析功能实施规划`](/D:/ProjectFloder/TS/bilibili_discovery/docs/兴趣分析功能实施规划.md) 拆解成可直接执行的开发任务。

默认目标：

- 以 `InterestContributionEvent` 为核心实现兴趣分析第一版
- 不复用 `Category`
- 不以 `WatchEvent` 作为兴趣分析事实表
- UI 只消费实现层聚合结果

---

## 1. 任务总览

推荐按 5 个阶段执行：

1. 数据模型与数据库配置
2. 原子实现层
3. 兴趣映射与归一化
4. 快照聚合与回填
5. UI 接入与验收

每个阶段完成后都应保证：

- TypeScript 可编译
- 数据库版本可正常升级
- 不破坏现有页面

---

## 4. 阶段三：兴趣映射与归一化

### Task 3.1 新建兴趣引擎目录

目标：

- 新建 `src/engine/interest/`

建议文件：

- `interest-types.ts`
- `interest-mapping.ts`
- `interest-scoring.ts`
- `interest-llm.ts`

完成标准：

- 纯算法和数据库读写解耦

### Task 3.2 实现固定兴趣主题常量

目标文件：

- `src/engine/interest/interest-types.ts`

需要完成：

- 定义第一阶段固定主题集合
- 提供主题名校验方法
- 提供默认主题初始化数据

完成标准：

- 主题集合不依赖 LLM
- 必须包含 `其他`

### Task 3.3 实现规则映射器

目标文件：

- `src/engine/interest/interest-mapping.ts`

需要完成：

- 编写 tag 到兴趣主题的规则映射
- 输出统一映射结果结构
- 支持一对多结果

建议优先覆盖：

- 游戏
- 音乐
- 科技
- 数码
- 知识
- 教程
- 动画
- 二次元
- 搞笑
- 鬼畜

完成标准：

- 已知常见 tag 可直接命中
- 未命中规则时返回空或待补充结果

### Task 3.4 实现 LLM 映射补洞器

目标文件：

- `src/engine/interest/interest-llm.ts`

依赖：

- 复用 `src/engine/llm-client.ts`

需要完成：

- 为单个 tag 构建 prompt
- 强制 LLM 输出固定主题集合内 JSON
- 解析、校验、标准化返回结构

完成标准：

- LLM 未配置时不会抛出致命错误
- 非法输出会回退

### Task 3.5 实现兴趣分数计算器

目标文件：

- `src/engine/interest/interest-scoring.ts`

需要完成：

- 根据观看时长、完播、行为类型计算贡献分数
- 提供纯函数接口
- 权重先写成常量配置

完成标准：

- 输入原始行为参数即可得到 `contributionScore`
- 无数据库依赖

### Task 3.6 实现兴趣分析聚合入口

目标文件：

- `src/database/implementations/interest-analysis.impl.ts`

需要完成：

- 初始化默认 `InterestTopic`
- 获取或生成 `TagInterestMapping`
- 将观看行为归一化成 `InterestContributionEvent`
- 将收藏行为归一化成 `InterestContributionEvent`

建议方法：

- `ensureDefaultTopics`
- `resolveTagMappings`
- `buildContributionEventsFromWatch`
- `buildContributionEventsFromFavorite`
- `recordWatchContribution`
- `recordFavoriteContribution`

完成标准：

- 给定一条行为输入，可生成并写入一组贡献事件

## 5. 阶段四：快照聚合与历史回填

### Task 4.1 实现快照重算方法

目标文件：

- `src/database/implementations/interest-analysis.impl.ts`

需要完成：

- 从 `InterestContributionEvent` 聚合 `7d`
- 从 `InterestContributionEvent` 聚合 `30d`
- 生成并写入 `InterestSnapshot`

建议方法：

- `rebuildSnapshotsForWindow`
- `rebuildRecentSnapshots`
- `getInterestSummary`

完成标准：

- 可输出按 `finalScore` 排序的兴趣列表
- 可生成占比 `ratio`

### Task 4.2 实现趋势计算

目标文件：

- `src/database/implementations/interest-analysis.impl.ts`

需要完成：

- 当前 `7d` 与上一个 `7d` 对比
- 当前 `30d` 与上一个 `30d` 对比
- 输出上升/下降/平稳

完成标准：

- 不必单独落库
- 可作为查询结果一部分返回

### Task 4.3 实现历史回填

目标文件：

- `src/database/implementations/interest-analysis.impl.ts`

需要完成：

- 扫描已有历史 tag
- 补齐 `TagInterestMapping`
- 扫描已有观看行为
- 生成历史 `InterestContributionEvent`
- 重建 `InterestSnapshot`

建议方法：

- `backfillTagMappings`
- `backfillContributionEvents`
- `rebuildAllSnapshots`

完成标准：

- 新功能接入后，旧数据能一次性补算

### Task 4.4 增加调试入口

推荐位置：

- `src/ui/test-tools/test-tools.ts`

需要完成：

- 增加按钮或调试入口，触发：
  - 初始化兴趣主题
  - 回填 tag 映射
  - 回填贡献事件
  - 重建快照

完成标准：

- 不需要用户手写脚本即可手动验证

## 10. 最小可交付版本

如果希望快速出第一版，最小闭环是：

1. 完成 `InterestTopic / TagInterestMapping / InterestContributionEvent / InterestSnapshot`
2. 能把观看行为转成贡献事件
3. 能按 `7d / 30d` 聚合快照
4. 能在页面展示兴趣 Top 列表

这个最小版本先不要求：

- LLM 必须接入
- 趋势必须很复杂
- 新页面必须完整设计

只要这四步通了，兴趣分析第一版就已经成立。
