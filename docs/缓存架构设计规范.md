# 缓存架构设计规范

## 1. 概述

本文档定义了缓存层的架构设计规范。缓存层是独立于UI实现的核心数据缓存系统,位于src/cache目录下,为整个应用提供统一的缓存服务。缓存层采用三层缓存架构,针对不同类型的数据使用不同的缓存策略,以优化性能和内存使用。

### 1.1 架构目标

- **性能优化**:通过缓存实现数据的快速访问,减少数据库访问次数
- **内存管理**:通过合理的缓存策略控制内存占用
- **数据一致性**:确保缓存数据与数据库数据的同步
- **可扩展性**:支持不同类型数据的缓存需求
- **独立性**:缓存层独立于UI实现,可被任何模块复用

### 1.2 架构层次

```
┌─────────────────────────────────────┐
│         查询层 (query)               │
│  - 调用缓存层接口                    │
└──────────────┬──────────────────────┘
               │ 访问缓存
┌──────────────▼──────────────────────┐
│         缓存层 (cache)               │
│  - 索引缓存 (Index Cache)            │
│  - 数据缓存 (Data Cache)             │
│  - 图像缓存 (Image Cache)            │
└─────────────────────────────────────┘
```

## 3. 缓存类型

### 3.1 索引缓存(Index Cache)

#### 3.1.1 用途

- 存储资源的轻量级索引信息
- 支持快速搜索和过滤
- 实现内存中的高效查询

#### 3.1.2 特性

- 只存储ID和关键字段
- 支持多维度索引(按收藏夹、标签、创作者等)
- 提供搜索和过滤接口
- 支持批量操作
- 使用LRU(最近最少使用)策略管理

#### 3.1.3 存储内容

- 资源ID
- 标题
- 创作者
- 标签
- 收藏夹ID
- 创建时间
- 更新时间

#### 3.1.4 使用场景

```typescript
// 搜索场景
1. 加载所有视频的索引(ID、标题、标签等)
2. 在内存中执行搜索和过滤
3. 获取匹配的视频ID列表
4. 根据ID从数据缓存获取完整数据
```

#### 3.1.5 更新机制

- 数据变更时不需要立即更新索引,只有初始化的时候会从数据库中获取原始数据,数据库更新不必强制让这里保持同步
- 批量更新接口支持高效更新
- 支持增量更新,避免全量重建

#### 3.1.6 容量控制

- 根据内存情况设置合理上限
- 使用LRU淘汰策略
- 提供容量监控接口

### 3.2 数据缓存(Data Cache)

#### 3.2.1 用途

- 存储资源的完整数据
- 支持按需加载和预加载
- 提供批量操作接口

#### 3.2.2 特性

- LRU淘汰策略
- 支持批量获取
- 实现预加载机制
- 提供缓存统计
- 按需加载,延迟获取完整数据

#### 3.2.3 存储内容

- 资源的完整数据对象
- 包括所有字段和关联数据

#### 3.2.4 使用场景

```typescript
// 分页加载场景
1. 查询层获取当前页的视频ID
2. 检查数据缓存,未命中则从数据库加载
3. 预加载下一页数据到缓存
4. 返回完整数据给查询层
```

#### 3.2.5 更新机制

- 首次访问时从数据库加载
- 提供批量预加载接口
- 支持手动刷新和失效

#### 3.2.6 容量控制

- LRU淘汰,保留最近使用的数据
- 根据内存情况动态调整
- 提供容量监控接口

### 3.3 图像缓存(Image Cache)

#### 3.3.1 用途

- 存储图像等大体积数据
- 实现懒加载机制
- 控制内存占用

#### 3.3.2 特性

- FIFO(先进先出)淘汰策略
- 固定容量限制
- 懒加载机制
- 优先级队列

#### 3.3.3 存储内容

- base64编码的图像数据
- 图像元数据(尺寸、类型等)

#### 3.3.4 使用场景

```typescript
// 图像加载场景
1. 监听滚动事件,检测图像进入可视区域
2. 检查图像缓存,未命中则加载
3. 加载完成后存入缓存
4. 超出容量限制时淘汰最早加载的图像
```

#### 3.3.5 更新机制

- 按需加载,延迟获取
- 支持优先级队列
- 提供手动清理接口

#### 3.3.6 容量控制

- FIFO淘汰,固定数量限制
- 懒加载机制,仅在需要时加载
- 提供容量监控接口

## 4. 缓存实现规范

### 4.1 基础缓存接口

```typescript
/**
 * 缓存接口
 * 定义缓存的基本操作
 */
export interface ICache<K, V> {
  /**
   * 获取缓存项
   * @param key 缓存键
   * @returns 缓存值
   */
  get(key: K): V | undefined;

  /**
   * 设置缓存项
   * @param key 缓存键
   * @param value 缓存值
   */
  set(key: K, value: V): void;

  /**
   * 批量设置缓存
   * @param entries 缓存项数组
   */
  setBatch(entries: Array<[K, V]>): void;

  /**
   * 删除缓存项
   * @param key 缓存键
   */
  delete(key: K): boolean;

  /**
   * 清空缓存
   */
  clear(): void;

  /**
   * 获取缓存统计
   * @returns 统计信息
   */
  getStats(): CacheStats;

  /**
   * 检查缓存是否存在
   * @param key 缓存键
   * @returns 是否存在
   */
  has(key: K): boolean;
}
```

### 4.3 资源缓存实现示例

```typescript
/**
 * 资源缓存类
 * 实现特定资源的缓存管理
 */
export class ResourceCache<K extends string | number, V> implements ICache<K, V> {
  private cache = new Map<K, V>();
  private stats: CacheStats = {
    size: 0,
    hits: 0,
    misses: 0,
    hitRate: 0
  };

  /**
   * 获取缓存项
   * @param key 缓存键
   * @returns 缓存值
   */
  get(key: K): V | undefined {
    const value = this.cache.get(key);
    if (value !== undefined) {
      this.stats.hits++;
    } else {
      this.stats.misses++;
    }
    this.updateHitRate();
    return value;
  }

  /**
   * 设置缓存项
   * @param key 缓存键
   * @param value 缓存值
   */
  set(key: K, value: V): void {
    this.cache.set(key, value);
    this.stats.size = this.cache.size;
  }

  /**
   * 批量设置缓存
   * @param entries 缓存项数组
   */
  setBatch(entries: Array<[K, V]>): void {
    entries.forEach(([key, value]) => this.set(key, value));
  }

  /**
   * 删除缓存项
   * @param key 缓存键
   * @returns 是否删除成功
   */
  delete(key: K): boolean {
    const result = this.cache.delete(key);
    this.stats.size = this.cache.size;
    return result;
  }

  /**
   * 清空缓存
   */
  clear(): void {
    this.cache.clear();
    this.stats.size = 0;
    this.stats.hits = 0;
    this.stats.misses = 0;
    this.stats.hitRate = 0;
  }

  /**
   * 获取缓存统计
   * @returns 统计信息
   */
  getStats(): CacheStats {
    return { ...this.stats };
  }

  /**
   * 检查缓存是否存在
   * @param key 缓存键
   * @returns 是否存在
   */
  has(key: K): boolean {
    return this.cache.has(key);
  }

  /**
   * 更新命中率
   */
  private updateHitRate(): void {
    const total = this.stats.hits + this.stats.misses;
    this.stats.hitRate = total > 0 ? this.stats.hits / total : 0;
  }
}
```

### 4.4 LRU缓存实现

```typescript
/**
 * LRU缓存实现
 * 使用最近最少使用策略管理缓存
 */
export class LRUCache<K extends string | number, V> extends ResourceCache<K, V> {
  private maxSize: number;
  private accessOrder: K[] = [];

  constructor(maxSize: number = 1000) {
    super();
    this.maxSize = maxSize;
  }

  override get(key: K): V | undefined {
    const value = super.get(key);
    if (value !== undefined) {
      this.updateAccessOrder(key);
    }
    return value;
  }

  override set(key: K, value: V): void {
    super.set(key, value);
    this.updateAccessOrder(key);
    this.evictIfNeeded();
  }

  override delete(key: K): boolean {
    const result = super.delete(key);
    if (result) {
      this.removeFromAccessOrder(key);
    }
    return result;
  }

  override clear(): void {
    super.clear();
    this.accessOrder = [];
  }

  private updateAccessOrder(key: K): void {
    const index = this.accessOrder.indexOf(key);
    if (index !== -1) {
      this.accessOrder.splice(index, 1);
    }
    this.accessOrder.push(key);
  }

  private removeFromAccessOrder(key: K): void {
    const index = this.accessOrder.indexOf(key);
    if (index !== -1) {
      this.accessOrder.splice(index, 1);
    }
  }

  private evictIfNeeded(): void {
    while (this.accessOrder.length > this.maxSize) {
      const keyToRemove = this.accessOrder.shift();
      if (keyToRemove !== undefined) {
        super.delete(keyToRemove);
      }
    }
  }
}
```

### 4.5 FIFO缓存实现

```typescript
/**
 * FIFO缓存实现
 * 使用先进先出策略管理缓存
 */
export class FIFOCache<K extends string | number, V> extends ResourceCache<K, V> {
  private maxSize: number;
  private insertionOrder: K[] = [];

  constructor(maxSize: number = 100) {
    super();
    this.maxSize = maxSize;
  }

  override set(key: K, value: V): void {
    if (!this.has(key)) {
      this.insertionOrder.push(key);
    }
    super.set(key, value);
    this.evictIfNeeded();
  }

  override delete(key: K): boolean {
    const result = super.delete(key);
    if (result) {
      const index = this.insertionOrder.indexOf(key);
      if (index !== -1) {
        this.insertionOrder.splice(index, 1);
      }
    }
    return result;
  }

  override clear(): void {
    super.clear();
    this.insertionOrder = [];
  }

  private evictIfNeeded(): void {
    while (this.insertionOrder.length > this.maxSize) {
      const keyToRemove = this.insertionOrder.shift();
      if (keyToRemove !== undefined) {
        super.delete(keyToRemove);
      }
    }
  }
}
```

## 5. 缓存监控

### 5.1 缓存统计信息

缓存层提供以下统计信息:

- 缓存大小(size)
- 命中次数(hits)
- 未命中次数(misses)
- 命中率(hitRate)
- 内存使用量(memoryUsage)

### 5.2 缓存性能监控

```typescript
/**
 * 缓存监控器
 */
export class CacheMonitor {
  private stats: Map<string, CacheStats> = new Map();

  /**
   * 记录缓存统计
   * @param cacheName 缓存名称
   * @param stats 统计信息
   */
  recordStats(cacheName: string, stats: CacheStats): void {
    this.stats.set(cacheName, stats);
  }

  /**
   * 获取缓存统计
   * @param cacheName 缓存名称
   * @returns 统计信息
   */
  getStats(cacheName: string): CacheStats | undefined {
    return this.stats.get(cacheName);
  }

  /**
   * 获取所有缓存统计
   * @returns 所有缓存统计信息
   */
  getAllStats(): Map<string, CacheStats> {
    return new Map(this.stats);
  }

  /**
   * 生成缓存使用报告
   * @returns 报告文本
   */
  generateReport(): string {
    let report = "缓存使用报告\n";
    report += "=".repeat(50) + "\n\n";

    for (const [name, stats] of this.stats.entries()) {
      report += `缓存: ${name}\n`;
      report += `  大小: ${stats.size}\n`;
      report += `  命中: ${stats.hits}\n`;
      report += `  未命中: ${stats.misses}\n`;
      report += `  命中率: ${(stats.hitRate * 100).toFixed(2)}%\n`;
      if (stats.memoryUsage !== undefined) {
        report += `  内存使用: ${this.formatBytes(stats.memoryUsage)}\n`;
      }
      report += "\n";
    }

    return report;
  }

  /**
   * 格式化字节数
   * @param bytes 字节数
   * @returns 格式化后的字符串
   */
  private formatBytes(bytes: number): string {
    if (bytes === 0) return "0 B";
    const k = 1024;
    const sizes = ["B", "KB", "MB", "GB"];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return `${(bytes / Math.pow(k, i)).toFixed(2)} ${sizes[i]}`;
  }
}
```

## 6. 错误处理

### 6.1 缓存错误处理

```typescript
/**
 * 缓存错误类
 */
export class CacheError extends Error {
  constructor(message: string, public readonly cause?: Error) {
    super(message);
    this.name = "CacheError";
  }
}

/**
 * 缓存操作错误处理示例
 */
export class SafeCache<K extends string | number, V> implements ICache<K, V> {
  private cache: ICache<K, V>;

  constructor(cache: ICache<K, V>) {
    this.cache = cache;
  }

  get(key: K): V | undefined {
    try {
      return this.cache.get(key);
    } catch (error) {
      console.error("[Cache] Get error:", error);
      return undefined;
    }
  }

  set(key: K, value: V): void {
    try {
      this.cache.set(key, value);
    } catch (error) {
      console.error("[Cache] Set error:", error);
      throw new CacheError("设置缓存失败", error as Error);
    }
  }

  setBatch(entries: Array<[K, V]>): void {
    try {
      this.cache.setBatch(entries);
    } catch (error) {
      console.error("[Cache] Batch set error:", error);
      throw new CacheError("批量设置缓存失败", error as Error);
    }
  }

  delete(key: K): boolean {
    try {
      return this.cache.delete(key);
    } catch (error) {
      console.error("[Cache] Delete error:", error);
      return false;
    }
  }

  clear(): void {
    try {
      this.cache.clear();
    } catch (error) {
      console.error("[Cache] Clear error:", error);
      throw new CacheError("清空缓存失败", error as Error);
    }
  }

  getStats(): CacheStats {
    try {
      return this.cache.getStats();
    } catch (error) {
      console.error("[Cache] Get stats error:", error);
      return {
        size: 0,
        hits: 0,
        misses: 0,
        hitRate: 0
      };
    }
  }

  has(key: K): boolean {
    try {
      return this.cache.has(key);
    } catch (error) {
      console.error("[Cache] Has error:", error);
      return false;
    }
  }
}
```

## 8. 最佳实践

### 8.1 缓存设计原则

1. **单一职责**:每个缓存类只负责一种类型的数据
2. **接口隔离**:使用明确的接口定义缓存操作
3. **依赖倒置**:依赖抽象接口而非具体实现
4. **开闭原则**:对扩展开放,对修改关闭

### 8.2 使用建议

1. 根据数据特性选择合适的缓存策略
2. 合理设置缓存容量
3. 定期监控缓存性能
4. 及时清理无用缓存
5. 避免缓存雪崩和缓存击穿

### 8.3 注意事项

1. 缓存不是数据库,不应长期存储数据
2. 注意缓存一致性,及时更新缓存
3. 避免缓存过大导致内存溢出
4. 处理缓存异常情况
5. 提供缓存失效机制

### 9.1 术语表

| 术语 | 定义 |
|-----|------|
| 索引缓存 | 存储资源ID和关键索引信息的缓存 |
