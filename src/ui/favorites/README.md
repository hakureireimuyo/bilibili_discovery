# 收藏视频模块

## 模块结构

```
favorites/
├── cache/      # 缓存层
├── query/      # 查询层
├── app/        # 应用层
└── index.ts    # 统一导出
```

## 查询缓存机制

### 三层架构

1. **应用层 (app/)**：处理页面交互和UI渲染
2. **查询层 (query/)**：负责数据查询和数据库操作
3. **缓存层 (cache/)**：提供高效的数据缓存

### 数据流程

```
用户请求 → 应用层 → 查询层 → 缓存层 → 数据库
```

### 缓存类型

#### 1. 视频索引缓存 (VideoIndexCache)
- 缓存视频关键信息：标题、创作者、标签、收藏夹ID
- 支持快速搜索：按标题、创作者名搜索
- 支持标签过滤：包含/排除标签
- 支持按收藏夹、创作者查询
- 页面关闭时自动清空

#### 2. 标签缓存 (TagCache)
- 缓存标签ID到名称的映射
- 使用LRU策略，最多缓存1000个标签
- 避免重复查询标签名称

#### 3. 封面缓存 (CoverCache)
- 缓存视频封面图片
- 使用LRU策略，总大小限制为10MB
- 自动淘汰最久未使用的封面

#### 4. 异步任务缓存 (AsyncTaskCache)
- 避免重复执行相同的异步任务
- 任务完成后自动清理

### 查询优化

#### 索引策略
- **收藏夹索引**：collectionId → videoIds
- **标签索引**：tagId → videoIds
- **创作者索引**：creatorId → videoIds

#### 预取机制
- 自动预取下一页数据
- 减少用户等待时间
- 提升浏览体验
