## 概述

Renderer 模块是介于数据层和UI层之间的中间层，负责将数据对象转换为网页元素，并实现分页和缓存功能。该模块提供了完整的渲染解决方案，使开发者能够专注于业务逻辑，而不必关心底层的渲染细节。

### 主要特点

- **关注点分离**：数据转换、分页管理、UI渲染各司其职
- **职责单一**：每个类只负责一个核心功能
- **单一真相源**：Book 作为配置的唯一真相源，确保数据一致性
- **观察者模式**：自动响应数据变化，无需手动轮询
- **缓存优化**：智能缓存策略，提高渲染性能

## 核心概念

### Book（数据层）

Book 是数据层的核心组件，负责：
- 与数据库交互
- 提供查询服务
- 管理数据索引
- 维护分页状态

**职责**：
- 数据的获取和存储
- 查询条件的处理
- 索引的更新和维护
- 分页状态的维护

### IElementBuilder（元素构建器）

元素构建器接口，负责将数据对象转换为网页元素。

**职责**：
- 将单个数据对象转换为网页元素
- 批量转换数据对象
- 可选地更新已有的网页元素

**实现要求**：
```typescript
interface IElementBuilder<TData, TElement> {
  buildElement(data: TData): TElement | Promise<TElement>;
  buildElements(dataList: TData[]): TElement[] | Promise<TElement[]>;
  updateElement?(element: TElement, data: TData): void | Promise<void>;
}
```

### RenderBook（渲染书）

RenderBook 是数据层和UI层之间的桥梁，负责：
- 将数据对象转换为网页元素
- 实现网页元素的分页显示
- 缓存网页元素以提高性能
- 响应数据变化并更新缓存

**职责**：
- 数据转换：通过 IElementBuilder 将数据对象转换为网页元素
- 分页管理：实现网页元素的分页显示
- 缓存管理：缓存最多三页的网页元素
- 生命周期管理：交付元素对象，但不管理其生命周期

### RenderList（渲染列表）

RenderList 是 UI 层的核心组件，负责：
- 从渲染书获取网页元素
- 将元素渲染成列表
- 提供翻页交互
- 管理所有元素

**职责**：
- 元素渲染：将网页元素渲染到容器中
- 翻页交互：提供上一页、下一页、跳转等功能
- 元素管理：管理所有元素的生命周期
- 更新通知：接收渲染书的更新通知

## 使用流程

### 完整使用流程

#### 步骤 1：准备数据层

首先需要创建 Book 实例，这是整个渲染流程的基础。

```typescript
import { bookManager } from './database/index.js';
import { CreatorRepository } from './database/index.js';
import { QueryService } from './database/index.js';

// 1. 创建 Repository 实例
const creatorRepo = new CreatorRepository();

// 2. 创建 QueryService 实例
const queryService = new QueryService(creatorRepo);

// 3. 创建 Book 实例
const queryCondition = {
  platform: 'bilibili',
  isFollowing: 0
};

const bookConfig = {
  repository: creatorRepo,
  queryService: queryService
};

const book = await bookManager.createBook(queryCondition, bookConfig);
```

**职责**：
- Repository：负责数据的持久化和查询
- QueryService：负责数据的查询逻辑
- Book：负责数据的索引和分页管理

#### 步骤 2：实现元素构建器

创建自定义的元素构建器，将数据对象转换为网页元素。

```typescript
import type { IElementBuilder } from './renderer/types.js';

class CreatorElementBuilder implements IElementBuilder<Creator, HTMLElement> {
  buildElement(data: Creator): HTMLElement {
    const element = document.createElement('div');
    element.className = 'creator-item';
    element.innerHTML = `
      <div class="avatar">
        <img src="${data.avatar}" alt="${data.name}">
      </div>
      <div class="info">
        <h3>${data.name}</h3>
        <p>粉丝数: ${data.fans}</p>
      </div>
    `;
    return element;
  }

  buildElements(dataList: Creator[]): HTMLElement[] {
    return dataList.map(data => this.buildElement(data));
  }
}
```

**职责**：
- 将数据对象转换为 DOM 元素
- 支持批量转换以提高性能
- 可选地支持元素更新

#### 步骤 3：创建渲染书实例

使用 Book 实例和元素构建器创建 RenderBook 实例。

```typescript
import { RenderBook } from './renderer/index.js';

// 创建渲染书实例
const renderBook = new RenderBook<Creator, HTMLElement>({
  book: book,
  elementBuilder: new CreatorElementBuilder(),
  maxCachePages: 3  // 最多缓存 3 页
});
```

**职责**：
- 数据转换：通过元素构建器将数据转换为元素
- 分页管理：实现元素的分页显示
- 缓存管理：缓存元素以提高性能

#### 步骤 4：创建渲染列表实例

使用 RenderBook 实例创建 RenderList 实例。

```typescript
import { RenderList } from './renderer/index.js';

// 实现自定义渲染列表
class CreatorRenderList extends RenderList<Creator, HTMLElement> {
  protected renderElements(elements: HTMLElement[]): void {
    this.container.innerHTML = '';
    elements.forEach(element => {
      this.container.appendChild(element);
    });
  }

  protected async deleteElement(element: HTMLElement, data: Creator): Promise<void> {
    // 从 DOM 中移除元素
    this.container.removeChild(element);
    // 更新数据库
    await creatorRepo.delete(data.id);
  }
}

// 创建渲染列表实例
const container = document.getElementById('creator-list') as HTMLElement;
const renderList = new CreatorRenderList({
  renderBook: renderBook,
  container: container,
  autoRender: false  // 初始时不自动渲染
});
```

**职责**：
- 元素渲染：将元素渲染到容器中
- 翻页交互：提供翻页功能
- 元素管理：管理元素的生命周期

#### 步骤 5：初始化并渲染

初始化渲染列表并渲染第一页。

```typescript
// 初始化渲染列表
await renderList.initialize();

// 渲染第一页
await renderList.renderCurrentPage();
```

#### 步骤 6：处理用户交互

响应用户的翻页操作。

```typescript
// 上一页
document.getElementById('prev-btn')?.addEventListener('click', async () => {
  await renderList.previousPage();
});

// 下一页
document.getElementById('next-btn')?.addEventListener('click', async () => {
  await renderList.nextPage();
});

// 跳转到指定页
document.getElementById('page-input')?.addEventListener('change', async (e) => {
  const page = parseInt((e.target as HTMLInputElement).value);
  await renderList.goToPage(page - 1);
});
```

#### 步骤 7：更新数据

当数据变化时，更新 Book 的索引。

```typescript
// 更新查询条件
const newCondition = {
  platform: 'bilibili',
  isFollowing: 1,
  keyword: 'search term'
};

// 更新索引（会自动通知 RenderBook）
await book.updateIndex(newCondition);

// 重新渲染当前页
await renderList.renderCurrentPage();
```

#### 步骤 8：销毁实例

不再需要时，销毁所有实例。

```typescript
// 销毁渲染列表（会自动取消注册监听器）
renderList.destroy();

// 销毁渲染书（会自动取消注册监听器）
renderBook.destroy();
```

### 实例创建时机

#### 创建顺序

必须按照以下顺序创建实例：

1. **Repository 和 QueryService**：数据层的基础
2. **Book 实例**：依赖 Repository 和 QueryService
3. **IElementBuilder 实例**：独立的转换逻辑
4. **RenderBook 实例**：依赖 Book 和 IElementBuilder
5. **RenderList 实例**：依赖 RenderBook

#### 销毁顺序

销毁时按照相反的顺序：

1. **RenderList 实例**：首先销毁 UI 层
2. **RenderBook 实例**：然后销毁渲染层
3. **Book 实例**：最后销毁数据层

## 最佳实践

### 1. 遵循创建顺序

始终按照依赖顺序创建实例，避免出现依赖未就绪的错误。

```typescript
// ✅ 正确的顺序
const repo = new CreatorRepository();
const queryService = new QueryService(repo);
const book = await bookManager.createBook(condition, config);
const elementBuilder = new CreatorElementBuilder();
const renderBook = new RenderBook({ book, elementBuilder });
const renderList = new CreatorRenderList({ renderBook, container });

// ❌ 错误的顺序
const renderList = new CreatorRenderList({ renderBook, container });  // renderBook 未创建
const renderBook = new RenderBook({ book, elementBuilder });  // book 未创建
```

### 2. 正确管理生命周期

确保在不再需要时销毁实例，避免内存泄漏。

```typescript
class MyComponent {
  private renderList: CreatorRenderList | null = null;
  private renderBook: RenderBook<Creator, HTMLElement> | null = null;

  async init() {
    // 创建实例
    this.renderBook = new RenderBook({ book, elementBuilder });
    this.renderList = new CreatorRenderList({ renderBook, container });
    await this.renderList.initialize();
  }

  destroy() {
    // 销毁实例
    if (this.renderList) {
      this.renderList.destroy();
      this.renderList = null;
    }
    if (this.renderBook) {
      this.renderBook.destroy();
      this.renderBook = null;
    }
  }
}
```

### 3. 使用接口而非具体类

使用接口而不是具体类，提高代码的灵活性和可测试性。

```typescript
// ✅ 使用接口
interface IRenderBook<TData, TElement> {
  getPage(page: number): Promise<RenderQueryResult<TElement>>;
  updateIndex(condition: QueryCondition): Promise<void>;
}

// ✅ 使用接口参数
function useRenderBook(renderBook: IRenderBook<Creator, HTMLElement>) {
  const result = await renderBook.getPage(0);
  // ...
}
```

### 4. 实现观察者模式

利用观察者模式自动响应数据变化，避免手动轮询。

```typescript
// RenderBook 自动注册为 Book 的监听器
const renderBook = new RenderBook({ book, elementBuilder });

// 当 Book 更新索引时，RenderBook 会自动收到通知
await book.updateIndex(newCondition);

// RenderBook 会自动清除缓存并通知 RenderList
// RenderList 会自动重新渲染当前页
```

### 5. 合理使用缓存

根据实际需求调整缓存策略。

```typescript
// 对于频繁翻页的场景，增加缓存页数
const renderBook = new RenderBook({
  book,
  elementBuilder,
  maxCachePages: 5  // 缓存 5 页
});

// 对于内存敏感的场景，减少缓存页数
const renderBook = new RenderBook({
  book,
  elementBuilder,
  maxCachePages: 1  // 只缓存当前页
});
```

## 常见问题

### Q1: 为什么必须按照特定顺序创建实例？

A: 因为组件之间存在依赖关系。RenderBook 依赖 Book，RenderList 依赖 RenderBook。如果不按照顺序创建，会导致依赖未就绪的错误。

### Q2: RenderBook 和 RenderList 的区别是什么？

A: 
- **RenderBook**：负责数据转换、分页管理和缓存，是数据层和UI层之间的桥梁
- **RenderList**：负责元素渲染、翻页交互和元素管理，是UI层的核心组件

### Q3: 如何处理数据更新？

A: 
1. 更新 Book 的索引：`await book.updateIndex(newCondition)`
2. RenderBook 会自动收到通知并清除缓存
3. RenderList 会自动收到通知并重新渲染

不需要手动触发更新，观察者模式会自动处理。

### Q4: 如何自定义元素渲染？

A: 
1. 继承 `RenderList` 类
2. 实现 `renderElements` 方法自定义渲染逻辑
3. 可选地实现 `deleteElement` 方法自定义删除逻辑

```typescript
class MyRenderList extends RenderList<Data, HTMLElement> {
  protected renderElements(elements: HTMLElement[]): void {
    // 自定义渲染逻辑
  }

  protected async deleteElement(element: HTMLElement, data: Data): Promise<void> {
    // 自定义删除逻辑
  }
}
```

### Q5: 如何优化性能？

A: 
1. **合理使用缓存**：根据场景调整 `maxCachePages`
2. **批量转换**：使用 `buildElements` 而不是多次调用 `buildElement`
3. **避免频繁更新**：利用观察者模式，避免手动轮询
4. **及时销毁实例**：不再需要时调用 `destroy()` 方法

### Q6: 如何处理错误？

A: 
1. 在 `renderElements` 和 `deleteElement` 中使用 try-catch
2. 在 `getPage` 和 `updateIndex` 中处理异常
3. 提供友好的错误提示

```typescript
class MyRenderList extends RenderList<Data, HTMLElement> {
  protected renderElements(elements: HTMLElement[]): void {
    try {
      this.container.innerHTML = '';
      elements.forEach(element => {
        this.container.appendChild(element);
      });
    } catch (error) {
      console.error('渲染失败:', error);
      this.container.innerHTML = '<div class="error">渲染失败</div>';
    }
  }
}
```

### Q6: 如何自定义渲染行为？

A: 继承 `RenderList` 并实现抽象方法：

```typescript
class MyRenderList extends RenderList<Data, Element> {
  protected renderElements(elements: Element[]): void {
    // 自定义渲染逻辑
    this.container.innerHTML = '';
    elements.forEach((el, index) => {
      // 添加动画效果
      el.style.animationDelay = `${index * 0.1}s`;
      this.container.appendChild(el);
    });
  }

  protected async deleteElement(element: Element, data: Data): Promise<void> {
    // 自定义删除逻辑
    if (confirm('确定删除吗？')) {
      this.container.removeChild(element);
      await repo.delete(data.id);
    }
  }
}
```
